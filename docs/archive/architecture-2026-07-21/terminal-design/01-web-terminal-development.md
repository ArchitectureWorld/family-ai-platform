# Family AI Platform：Web 终端开发方案

## 1. 终端定位

Web 端是 `family-ai-platform` 的全功能个人工作台和家庭 AI 管理入口。

核心定位：

```text
完整工作
复杂内容处理
文件操作
系统管理
```

Web 不是移动端的简单放大版，而是信息密度最高、工作能力最完整的终端。

---

## 2. 核心用户场景

- 与个人助理进行长文本交流；
- 处理复杂 Work Conversation；
- 阅读、上传和分析文件；
- 查看任务、决定和执行状态；
- 管理家庭成员、设备和入口绑定；
- 查看 Agent 与终端运行状态；
- 从 Chat 将事项转入 Work；
- 多栏并行查看内容。

---

## 3. 页面结构

建议一级结构：

```text
Chat
Work
管理
```

其中“管理”仅对具备权限的家庭管理员显示。

### Chat 页面

建议结构：

```text
左侧：近期 Work / 提醒 / 快速入口
中间：HomeChatStream
右侧：上下文、文件、任务或状态
```

第一版可以先采用单主栏加可收起侧栏。

### Work 页面

```text
左侧：Conversation 列表
中间：当前 Conversation
右侧：文件、任务、决定、执行记录
```

### 管理页面

```text
家庭成员
个人助理绑定
设备
入口绑定
DIY 终端
权限
运行状态
```

---

## 4. Chat 体验

Web Chat 必须连接统一 `HomeChatStream`。

支持：

- 文本输入；
- 图片和文件拖拽；
- 多段内容粘贴；
- 消息引用；
- 消息片段“转为 Work”；
- 展示来自其他终端的消息；
- 查看每日归档分界；
- 搜索历史 Chat；
- 查看最近 Work 状态。

不建议第一版在 Chat 中堆叠过多仪表盘，核心仍应是自然对话。

---

## 5. Work 体验

Web 是 Work 的主终端。

第一版需要支持：

- 新建 Conversation；
- Conversation 列表；
- 搜索；
- 归档；
- 重命名；
- 跨端继续；
- 上传文件；
- 查看 Context Summary；
- 查看执行过程；
- 查看阶段结论；
- 查看待用户确认项。

后续增强：

- 多 Conversation 分栏；
- 文件与对话并排；
- 任务看板；
- Agent Team 协作可视化；
- 项目空间；
- 画布式工作台。

---

## 6. 身份绑定

### 普通用户

```text
账号登录
→ 验证身份
→ EntryBinding
→ Person
→ 个人助理
```

首次进入若未绑定，可进入手机号认领流程。

### 家庭管理员

管理员权限属于 Person 的角色权限，不能通过访问特定 URL 隐式获得。

---

## 7. 跨端接续

Web 首页应显示：

```text
最近活动
- 刚刚在 iPhone Chat 中继续了某项讨论
- 某个 Work Conversation 有新结果
```

Web 打开后默认进入上次本设备使用的页面；若无本地记录，进入 Chat。

不得因为用户刚在手机上打开某个 Work，就强制 Web 跳转。

---

## 8. 文件处理

Web 是文件能力最完整的终端。

第一版支持：

- 上传；
- 下载；
- 预览；
- 关联到 Chat 或 Conversation；
- 删除；
- 查看处理状态。

文件归属必须明确：

```text
Person Private
Family Shared
Conversation Scoped
Temporary
```

---

## 9. 通知

Web 支持：

- 站内通知；
- 浏览器通知；
- 任务执行完成；
- Work 等待确认；
- 设备绑定事件；
- 安全事件。

浏览器通知需要用户明确授权。

---

## 10. 管理端职责

管理端第一版包括：

- 预创建家庭成员；
- 配置手机号；
- 配置个人助理；
- 查看待认领状态；
- 查看已绑定入口；
- 解绑设备；
- 注册 DIY 设备；
- 设置设备归属；
- 冻结成员或设备。

不建议第一版加入复杂权限矩阵编辑器。

---

## 11. 弱网与异常

Web 需要处理：

- 草稿自动保存；
- 消息发送中；
- 断线重连；
- 文件上传续传或明确重试；
- 重复消息去重；
- Conversation 被其他终端归档；
- 登录失效。

---

## 12. 第一版范围

### 必须

- 登录与绑定；
- Chat；
- Work Conversation；
- 多端同步；
- 文件上传；
- Chat 转 Work；
- 管理员成员与设备管理；
- 终端状态显示。

### 暂缓

- 多窗口复杂布局；
- 项目画布；
- 完整任务看板；
- 多 Agent 可视化；
- 高级 BI 仪表盘；
- 插件市场。

---

## 13. 体验验收

- 登录后正确进入本人 Chat；
- Web 与移动端消息实时一致；
- 能从 Chat 创建 Work；
- 能在 Work 中继续移动端 Conversation；
- 文件能够正确归属；
- 普通成员看不到管理员功能；
- 设备解绑后旧会话失效；
- 弱网恢复后无重复消息。
