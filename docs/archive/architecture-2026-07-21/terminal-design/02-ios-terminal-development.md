# Family AI Platform：iOS 终端开发方案

## 1. 终端定位

iOS 是第一优先级的个人随身入口。

核心定位：

```text
随时可用
默认 Chat
轻量 Work
通知与确认
现场输入
```

iOS 不追求复制 Web 的全部复杂能力，而是保证个人助理始终可触达。

---

## 2. 一级页面

建议：

```text
Chat
Work
我的
```

第一版可以使用：

```text
Chat
Work
```

并将账号和设备设置放入顶部入口。

默认启动进入 Chat。

---

## 3. Chat 体验

Chat 连接用户唯一的 `HomeChatStream`。

第一版支持：

- 文字；
- 语音；
- 图片；
- 相机拍摄；
- 基础文件；
- 消息同步；
- 推送提醒；
- 快速确认；
- 复制和引用；
- 转为 Work。

启动时不需要选择 Agent 或 Session。

用户从 Web 切换到 iOS 后，应能立即看到最新 Chat 消息。

---

## 4. Work 体验

iOS Work 用于：

- 查看 Conversation 列表；
- 打开已有 Conversation；
- 简单继续讨论；
- 上传现场图片；
- 查看 Agent 执行结果；
- 完成确认、选择和批准；
- 查看阶段结论；
- 接收等待用户处理的事项。

第一版不需要：

- 复杂多栏编辑；
- 大型文件深度处理；
- 多 Conversation 并排；
- 完整管理后台。

---

## 5. 首页连续性

打开 App 后：

```text
默认进入 Chat
```

同时可显示轻量提示：

```text
你刚才在 Web 端处理：
family-ai-platform 多终端架构
```

用户可选择继续 Work，但不能强制跳转。

---

## 6. 身份绑定

首次绑定：

```text
输入手机号
→ 验证码
→ 确认家庭成员
→ 建立设备与账号绑定
→ 进入 Chat
```

后续：

```text
App 登录态 / 设备凭证
→ EntryBinding
→ Person
```

需要支持：

- 更换设备；
- 退出登录；
- 解绑；
- 账号恢复；
- 设备丢失后的远程失效。

---

## 7. 通知策略

iOS 是主要提醒终端。

通知类型：

- 明确截止日期；
- 会议；
- 需要确认；
- Agent 执行完成；
- Work 等待决策；
- 家庭重要事项；
- 安全事件。

通知点击后必须打开正确目标：

```text
Chat
或
指定 Conversation
```

---

## 8. 语音体验

第一版语音能力可以采用：

```text
按住说话 / 点击录音
→ 语音转文字
→ 用户可确认
→ 发送
```

后续再扩展连续对话和实时语音。

语音输出默认不自动播放，除非用户开启相关设置或处于语音模式。

---

## 9. 相机与现场输入

iOS 的优势之一是现场采集。

应支持：

- 拍照发给 Chat；
- 拍照发给指定 Work；
- 扫描文档；
- 录制短语音；
- 分享其他 App 内容到 family-ai-platform。

分享入口必须让用户确认目标：

```text
发送到 Chat
发送到某个 Work Conversation
```

---

## 10. 离线与弱网

第一版至少支持：

- 本地保存未发送草稿；
- 发送队列；
- 恢复网络后重试；
- 去重；
- 清晰显示未同步状态；
- 不在离线状态下假装 Agent 已收到。

完整离线 Agent 不属于第一版。

---

## 11. 隐私

个人设备可保持登录，但应支持：

- Face ID / 系统认证保护；
- 隐藏通知内容；
- 私密 Conversation；
- 设备丢失远程解绑；
- 敏感操作二次确认。

---

## 12. 第一版范围

### 必须

- 手机号认领；
- 账号与设备绑定；
- Chat；
- Work 列表；
- Conversation 继续；
- 文字、语音、图片；
- 推送；
- Chat 转 Work；
- 多端同步；
- 基础安全。

### 暂缓

- 连续实时语音；
- 完整离线模式；
- iPad 专业多栏；
- 高级自动化；
- 系统级深度设备联动；
- 完整 Apple Watch 入口。

---

## 13. 体验验收

- 安装并认领后直接进入正确个人 Chat；
- 与 Web 端消息一致；
- 推送点击进入正确对象；
- 可以将 Chat 内容转入 Work；
- Work 中能完成确认和轻量继续；
- 弱网下不重复消息；
- 退出或解绑后无法访问私人数据；
- 图片和语音能正确标记来源设备。
